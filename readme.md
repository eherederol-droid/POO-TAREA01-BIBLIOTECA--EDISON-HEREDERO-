## Breve Resumen de la estructura del caso de estudio

El proyecto se desarrolla siguiendo cuatro fases fundamentales. En la fase de requerimientos, se identifican las necesidades del sistema, tales como el registro de personas y estudiantes, la administración de libros y el control de préstamos y devoluciones. En la fase de análisis, se determinan las entidades principales del sistema (Persona, Estudiante, Libro, Préstamo y Biblioteca) y se establecen sus relaciones.

Durante la fase de diseño, se elabora un diagrama de clases utilizando PlantUML, donde se definen los atributos, métodos, visibilidad y relaciones entre las clases, incluyendo herencia, asociación y cardinalidad. Finalmente, en la fase de desarrollo, se implementa el sistema en Python, aplicando principios como encapsulamiento, modularidad y reutilización de código.

Este sistema permite simular el funcionamiento básico de una biblioteca, facilitando la gestión y organización de la información de manera clara, estructurada y eficiente.
